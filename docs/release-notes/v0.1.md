## 0.1.0 — 2026-05-16

### Feature: Initial release of the Claude Code Starter

**Value:** A fork-and-go Next.js template that gives a new web app authentication, an admin shell, roles and permissions, TOTP 2FA, environment feature flags, an audit log, and release notes out of the box — wired to Neon Postgres and ready to deploy on Vercel. Also serves as a teaching artifact for working with Claude Code: the `.claude/` workspace, the SDLC pipeline, and the periodic-review cadence are part of the deliverable.

#### What's New

- **Authentication.** NextAuth 5 (beta) with Google OAuth and a credentials flow. JWT sessions carry the user's roles, features, and 2FA state so every downstream check is one read away.
- **TOTP 2FA.** Enrolment with a QR code, verification with recovery codes, and a trusted-device cookie for the "remember this browser" affordance. Secrets are AES-GCM-encrypted at rest under `AUTH_TOTP_ENCRYPTION_KEY`.
- **Roles and permissions.** A `roles` ↔ `features` ↔ `users` model. `FEATURES` in `src/lib/permissions.ts` is the static catalog; `hasFeature()` is the runtime check. The seed script binds the admin role to every feature on a fresh install.
- **Feature flags.** A `feature_flags` table with an environment-level toggle and rollout percent. `isFlagEnabled(key)` is the runtime check. Distinct from permissions on purpose.
- **Admin shell.** `/admin` with subpages for users, roles, flags, docs, and 2FA management. Gated by the `admin.dashboard` feature.
- **Audit log.** Append-only `audit_events` table for security-sensitive actions (role changes, flag toggles, TOTP resets, deactivations).
- **Release notes viewer.** The admin docs page renders this file and its successors from `docs/release-notes/vX.Y.md`.
- **Edge route gate.** `src/middleware.ts` and `src/proxy.ts` enforce the auth + 2FA gate at the edge so unauthenticated traffic never reaches a page render.
- **Seed script.** `scripts/seed.ts` creates the admin and member roles, seeds every feature in `FEATURE_CATALOG`, binds the admin role to all of them, and registers a demo feature flag.

| Feature key | Required for |
|-------------|--------------|
| `admin.dashboard` | Reach the `/admin` shell. |
| `admin.users` | View and edit users under `/admin/users`. |
| `admin.flags` | Toggle environment feature flags. |
| `admin.release_notes` | Read release notes from the admin docs page. |

#### Routes

- `GET /` — Public landing page.
- `GET /signin` — Sign-in (Google + Credentials).
- `GET /signin/totp` — TOTP enrolment + verification.
- `GET /access-pending` — Landing for authenticated users with no roles.
- `GET /admin` — Admin shell.
- `GET /admin/users` — Manage users.
- `GET /admin/flags` — Manage feature flags.
- `GET /admin/docs` — Read release notes and other admin docs.
- `GET /admin/2fa` — Manage 2FA enforcement and reset enrolments.

#### Environment Variables Required

| Variable | Purpose |
|----------|---------|
| `DATABASE_URL` | Neon pooled connection string (host includes `-pooler`). |
| `DATABASE_URL_UNPOOLED` | Neon direct connection for Drizzle Kit. |
| `AUTH_SECRET` | NextAuth JWT signing key. Generate with `openssl rand -base64 32`. |
| `AUTH_URL` | Public origin (e.g., `https://app.example.com`). |
| `AUTH_GOOGLE_ID` / `AUTH_GOOGLE_SECRET` | Google OAuth credentials. |
| `AUTH_TOTP_ENCRYPTION_KEY` | 32-byte key for encrypting TOTP secrets at rest. |
