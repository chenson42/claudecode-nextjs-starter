## 0.2.0 — 2026-05-16

### Feature: Per-user 2FA bypass + force-reset

**Value:** Admins can now exempt a specific user from the TOTP gate (typically the seeded test account for demoing) and can force-reset a user's enrolled TOTP secret + recovery codes, all from a new `/admin/users/[id]` detail page. The seeded admin already lands with `twoFactorRequired: false` so a fresh fork can sign in and reach `/admin` immediately, without standing up a TOTP authenticator.

#### What's New

- **`/admin/users/[id]` detail page.** A per-user surface for the existing user-management actions; gated by an explicit `hasFeature(FEATURES.ADMIN_USERS)` check on the page (the edge proxy only covers `admin.dashboard`).
- **TwoFactorCard.** Inline status messages (no toast library in the starter), a checkbox to flip `twoFactorRequired`, and a shadcn `AlertDialog` confirmation before force-reset.
- **Self-disable block.** `setTwoFactorRequired` rejects when actor and target are the same user trying to disable — admins can't lock themselves out of their own 2FA.
- **Visual badge.** Users with `twoFactorRequired: false` show a "2FA exempt" pill on `/admin/users` so the bypass is never invisible.

### Feature: `AUDIT_ACTIONS` typed catalog

**Value:** Audit strings used to be inline literals scattered across four `actions.ts` files — easy to mistype, easy to drift, impossible to find via "go to references". They now live in a single `src/lib/audit.ts` catalog with frozen string values and a unit test that pins both shape and exact values.

- Migrated six existing audit strings (`user.role.assigned`, `user.role.revoked`, `flag.toggled`, etc.) into typed exports.
- Two new entries: `USER_TWO_FACTOR_REQUIRED_CHANGED`, `USER_TWO_FACTOR_FORCE_RESET`.
- String values are guaranteed unchanged from v0.1 so the audit log keeps a stable identifier set across the upgrade.

### Feature: End-to-end test infrastructure

**Value:** The starter previously documented Vitest + Playwright as recommended add-ons. Both now ship configured. A fork can write its first e2e test without thinking about config.

- `@playwright/test` ^1.60 in devDependencies; chromium-only browser install.
- `playwright.config.ts` at repo root: single project, loads `.env.local`, baseURL from `E2E_BASE_URL`.
- New script: `npm run test:e2e`. The dev server must be running (`npm run dev`) before invoking — Playwright does not spawn it.
- First spec: `e2e/admin-login.spec.ts` exercises the seeded-admin sign-in flow and walks into every admin subpage, asserting each renders without a runtime error.

### Bug fix: Admin sign-in dropped users on `/access-pending`

**Symptom:** Sign in as the seeded admin → land on `/access-pending` instead of `/admin`. The JWT callback correctly stamped `roles: ['admin']` + 4 features into the token, but the proxy still saw an empty session.

**Cause:** The `session` callback that projects JWT claims onto `session.user` was defined only in `src/auth.ts`. The shared `authConfig` in `src/lib/auth/config.ts` — which is what the edge runtime imports — had no session callback, so `edgeAuth()` returned a default session with no `roles` or `features`. The proxy then ran its admin check against an empty array and bounced every authenticated user.

**Fix:** Extracted the projection into a pure `projectJWTOntoSession` function in `src/lib/auth/session-projection.ts` and wired the shared `authConfig` to it. Both runtimes now use the same projection. A regression unit test pins the projection's contract; the Playwright spec catches the wiring end-to-end.

### Bug fix: `/admin/users` threw on the role-lookup query

**Symptom:** Navigating to `/admin/users` returned a runtime error from Drizzle: "Failed query ... ANY($1)".

**Cause:** The page used a raw ``sql`${userRoles.userId} = ANY(${userIds})` `` template. The Neon HTTP driver doesn't bind a JavaScript array to a Postgres array parameter through that interpolation.

**Fix:** Replaced with Drizzle's `inArray(userRoles.userId, userIds)` helper. Caught by the expanded e2e spec that now opens every admin subpage.

### Internal

- **Landing page reflects session state.** `src/app/page.tsx` now awaits `auth()`. Signed-out visitors see the existing "Sign in" / "Go to admin" pair. Signed-in visitors get a "Welcome back, {name}." line and a "Sign out" button (server action calling `signOut({ redirectTo: "/" })`) in place of the "Sign in" link.
- **Typography plugin.** Added `@tailwindcss/typography` and wired it in via `@plugin "@tailwindcss/typography"` in `globals.css`. The `prose` classes on the admin docs viewer were inert without it — every heading, list, and code span rendered as flat unstyled text. This is the fix for v0.1's "release notes are ugly" gap that surfaced only once v0.2's richer structure made it obvious.
- **Sign-in form a11y.** `<label>` elements now use `htmlFor` linked to input `id`s.
- **`.gitignore`.** Added `coverage/`, `test-results/`, `playwright-report/`.
- **Removed debug logging** from `src/auth.ts` JWT callback (leftover from the access-pending bug hunt).
